# Чеклист публикации

Перед публикацией:

1. Проверить изменения:
   `git status`
2. Сохранить их в историю:
   `git add .`
   `git commit -m "Что изменено"`
3. По желанию сделать метку перед релизом:
   `git tag release-YYYY-MM-DD-HHMM`
4. Отправить изменения в GitHub:
   `git push`
   `git push --tags`

Публикация:

1. Запустить:
   `.\scripts\publish-theme.ps1`
2. Ввести пароль хостинга, если будет запрос.
3. Дождаться сообщения об успешной выкладке.

Проверка после публикации:

1. Открыть главную страницу.
2. Открыть 2-3 важные страницы сайта.
3. Проверить меню, футер, формы и изображения.
4. Если есть ошибка, сразу откатиться на прошлую рабочую версию.

Если что-то сломалось:

1. Вернуть прошлый рабочий коммит или тег.
2. Снова запустить:
   `.\scripts\publish-theme.ps1`
