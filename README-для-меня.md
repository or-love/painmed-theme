# Painmed Theme

Это папка только с темой WordPress `painmed`.

Главная идея:
- `painmed/` = сами файлы темы
- Git = машина времени для файлов
- GitHub = главное место хранения темы
- Beget = место публикации

## Какая схема выбрана

Есть 2 простых варианта:

1. Только `main` + частые коммиты + метки перед публикацией
2. `main` + временные ветки для больших правок

Для вас выбран вариант 1 как основной.
Почему: он проще, понятнее и даёт лёгкий откат.

Если правка большая и страшно, можно временно сделать отдельную ветку позже.

## Что лежит в этой папке

- `painmed/` - живая тема
- `backups/` - локальные резервные копии
- `scripts/` - простые скрипты для публикации
- `README-для-меня.md` - эта инструкция
- `DEPLOY-CHECKLIST.md` - короткий чеклист перед публикацией

## Как работать дальше

1. Открыть эту папку.
2. Проверить, что правите файлы только внутри `painmed/`.
3. Перед важной правкой сделать коммит.
4. После правок проверить сайт локально глазами по файлам.
5. Сделать коммит с понятным сообщением.
6. Отправить изменения в GitHub.
7. Опубликовать тему на Beget.
8. Проверить сайт.

## Базовые команды

Проверить состояние:

```powershell
git status
```

Сохранить изменения в историю:

```powershell
git add .
git commit -m "Коротко что изменено"
```

Отправить в GitHub:

```powershell
git push
```

## Как публиковать тему

Основной путь:

```powershell
.\scripts\publish-theme.ps1
```

Скрипт:
- попросит доступ по SSH/SFTP;
- создаст резервную копию текущей темы на Beget;
- загрузит новую версию темы;
- заменит тему на сервере.

Перед первым запуском убедитесь, что у FTP-аккаунта на Beget включён SSH-доступ.

Перед первым запуском нужно один раз создать файл:

```powershell
Copy-Item .\scripts\deploy.config.example.ps1 .\scripts\deploy.config.ps1
```

## Как откатиться, если сайт сломался

Самый простой путь:

1. Найти последнюю рабочую точку:

```powershell
git log --oneline --decorate
```

2. Вернуть файлы темы из рабочего коммита или тега:

```powershell
git restore --source <commit-or-tag> painmed
```

3. Зафиксировать откат:

```powershell
git add .
git commit -m "Откат к рабочей версии"
```

4. Снова опубликовать тему:

```powershell
.\scripts\publish-theme.ps1
```

## Аварийный запасной путь

Если скрипт публикации недоступен:
- упаковать папку `painmed` в zip;
- загрузить её на хостинг вручную;
- распаковать вместо текущей темы.

Этот способ аварийный, не основной.
